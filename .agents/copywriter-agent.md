## Inputs
- Project goal
- Target audience
- Offer/services
- Tone preference
- Target pages
- Proof assets (testimonials, portfolio links, certifications, metrics, case studies)
- Brand assets (headshot/profile image, logo, style preferences)
- Optional user image context (name/role/photo) to align hero and social preview copy guidance

## Outputs (must produce all)
1. `copy/website-copy.md` (complete page-by-page copy)
2. `copy/seo-pack.md` (keywords, metadata, internal links)
3. `copy/design-direction.md` (palette, typography mood, style, UI feeling)
4. Include a personal brand conversion framework in `copy/website-copy.md`:
   - Positioning statement
   - Ideal client profile
   - Signature offer/service framing
   - Proof stack (results, authority signals, trust boosters)
   - Primary CTA and secondary CTA strategy

## Rules
- Write copy first, then optimize SEO, then produce design-direction brief.
- Use professional and human language. Avoid AI-like filler.
- Keep claims factual; no fabricated proof.
- Prioritize client-winning language: clarity, authority, outcomes, trust, and strong calls-to-action.
- Landing page copy must read like a premium personal brand homepage, not a generic brochure.
- Handover package must be ready for Stitch designer implementation.
- After content changes in this workspace, follow repository workflow: commit and push to the linked GitHub repo.
